# Jetson Orin Nano deployment requirements
# JetPack 6.0+ (CUDA 12.2) recommended for best AWQ compatibility
#
# SETUP SEQUENCE (run on Jetson):
# 1. Flash JetPack 6.0 via SDK Manager on host machine
# 2. On Jetson: sudo apt install python3.10 python3.10-venv python3.10-dev
# 3. python3.10 -m venv ~/venvs/openvla_deploy
# 4. source ~/venvs/openvla_deploy/bin/activate
#
# 5. Install Jetson-specific PyTorch wheel (NOT from standard PyPI — it has no GPU):
#    Check https://forums.developer.nvidia.com/t/pytorch-for-jetson/72048 for
#    the current JetPack 6 wheel. As of JetPack 6.0:
#      wget https://developer.download.nvidia.com/compute/redist/jp/v60dp/pytorch/torch-2.2.0a0+6a974be.nv23.11-cp310-cp310-linux_aarch64.whl
#      pip install torch-2.2.0a0+...linux_aarch64.whl
#    CUDA availability check: python -c "import torch; print(torch.cuda.is_available())"
#    Expected: True (if False, you installed the wrong wheel)
#
# 6. Install jetson-stats for power monitoring:
#    sudo pip3 install jetson-stats==4.2.7
#    sudo systemctl restart jtop.service
#
# 7. Install remaining requirements:
#    pip install -r jetson_requirements.txt
#
# 8. For LIBERO rollouts on Jetson (optional — only needed for full eval, not latency bench):
#    Install libero from git (same as training env)
#    Note: LIBERO sim is CPU-bound on Jetson — expect ~1-2 FPS sim speed.
#    Set MUJOCO_GL=egl for headless; osmesa works but is ~3x slower.
